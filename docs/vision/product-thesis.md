# DigiD product thesis

## One-sentence thesis

DigiD is a verifiable communications identity and provenance layer for humans, agents, and organizations across voice, video, email, messaging, and social channels.

## Core problem

Digital communication has a sender-trust problem.

Existing systems can show an account name, a phone number, an email address, or a profile, but that is not enough anymore. Receivers increasingly need to know:
- whether the sender is a real verified human
- whether the sender is an AI agent
- whether the sender acts under an organization
- whether the message or media was signed by the claimed party
- whether the communication has a trustworthy provenance trail

## The shift DigiD responds to

The internet was built for account identity.
The next layer must handle communication identity.

That means the system should not only answer:
- who owns this account?

It should answer:
- who originated this communication?
- what kind of entity are they?
- what proof supports that claim?
- what level of confidence should the receiver have?

## Product framing

DigiD should initially be framed as:
- a communications authenticity layer
- a sender verification protocol
- a provenance and trust-signaling system

It should not initially be framed as:
- a mandatory universal identity registry
- a replacement for government identity systems
- a surveillance platform for tracking all humans by default

## Initial trust-state model

DigiD should make it easy to distinguish between:
- verified human
- verified agent
- verified organization
- pseudonymous persistent identity
- unverified identity
- revoked or disputed identity

## Visible user value

DigiD only matters if trust is legible.

Receivers should be able to see a clear trust state in context:
- during a call
- beside a voice note
- beside a video
- beside a message or email
- beside a social profile or post

## Initial market wedge

The first strong commercial wedge is likely not "identity for everyone."
It is likely one of:
- verified AI communications
- enterprise agent identity and delegation
- signed media provenance across voice and video
- trust infrastructure for organizations sending messages via humans and agents

## Strategic principle

Start narrow enough to ship.
Design broad enough to become a standard.
